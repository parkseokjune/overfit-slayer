# 운영 규칙서 (2026-06-11 제정 — 외부 리뷰 채택)

> **운영용 한 줄 원칙**
> 90일 전에는 학습하지 말고 관찰한다.
> 90일 후에도, 좋아 보이는 변화가 아니라 충분히 검증된 변화만 교체한다.
> 잔고·체결·드리프트가 흔들리면 수익보다 생존을 우선한다.

이 시스템의 목표는 "자동으로 똑똑해지는 시스템"이 아니라 **"자동으로 망가지지 않는 시스템"**이다.
수익성 목표 = CAGR 최대화가 아니라 **드로다운 통제 후 기대수익의 안정적 누적**.

## 자가학습 교체 허용 조건표 (코드: src/self_learn.py `_policy_block` + `decide`)

| 항목 | 허용 조건 | 구현 |
|---|---|---|
| 라이브 경과 | 90일 이상 | `_policy_block` 관측 모드 |
| 드리프트 (평상시) | 정상이면 보수 유지 — 교체 불가 | `_policy_block` 안정 상태 차단 |
| 드리프트 (열화 시) | WARNING: 마진 0.15 / CRITICAL: 마진 0.05로 교체 가능 | `_current_margin` |
| OOS 개선폭 | 현재 대비 Sharpe +0.15 이상 | `IMPROVE_MARGIN` |
| plateau | 인접 이웃(맨해튼 1) 중앙값 > 0 | `_neighbors` |
| 교체 빈도 | 직전 2회 연속 교체 시 쿨다운 | `_policy_block` |
| 북당 변경 수 | 1개만 | `self_learn` 루프 구조 |
| 학습 대상 | 전략 파라미터만 (레버리지/스탑 고정) | `GRIDS` 범위 |
| 체결 상태 | 포지션 불일치 시 학습 보류 | `_policy_block` recon_block |
| 잔고 상태 | 0이면 학습·진입 모두 차단 | `reconcile` + `_policy_block` |

## 의사결정 규칙

- **A. 무조건 유지**: 라이브 90일 미만 / 드리프트 정상 / 개선폭 <0.15 / 쿨다운 / 체결·잔고 이상
- **B. 제한적 교체**: 90일+ & 드리프트 WARNING/CRITICAL & plateau 통과 & 개선폭 충족 & 쿨다운 아님
- **C. 즉시 중단·축소**: 포지션 불일치 반복 / 잔고 0 / CRITICAL 반복 / PSR 장기 하락 / MDD -40% (킬스위치)

## 90일 관측 종료 체크리스트 (자동 평가: `python -m src.gate_check`)

1. **데이터 충분성**: 라이브 90일+, 결측 없는 일별 곡선, 리셋/오류/재시작 기록(epoch 마커), 드리프트 30일+ 연속
2. **체결 정합성**: 장부-거래소 불일치 반복 없음, 잔고 0 미발생(발생 시 즉시 복구), maker/폴백 비율 정상, 실측 슬리피지가 가정(0.05%) 근방
3. **성과 신뢰성**: PSR > 0.5 지속, 드리프트 분위 ≥ 25%, 최근 30일 CRITICAL 반복 없음, 북별 손익 편중 없음
4. **포트폴리오 건전성**: MDD 허용 한도 내, 북 간 상관 유지·하락, 북별 역할(CONTRIBUTION.md) 실측 일치, 단일 북 악화가 전체를 끌어내리지 않음
5. **운영 안정성**: state/equity/revalidation/drift 파일 일관 생성, 반복 실패 사이클 없음, 자가학습이 유지/교체를 정책대로 기록, 쿨다운 정상 작동

**전 부문 통과 시에만** B규칙 단계 진입. 미통과 항목이 있으면 관측 연장.

## 월간 운영 체크리스트 (자동: 매월 1일 러너 실행 `src/monthly_check.py` → results/monthly_report_YYYY-MM.json)

> **월간 한 줄 원칙**: 월간 운영은 성과 추적이 아니라 생존 점검이다.
> 바꿀 수 있다고 해서 바꾸지 말고, 바꿔도 되는 조건이 전부 맞을 때만 바꾼다.
> 이 시스템은 매달 바꾸는 시스템이 아니라, **매달 확인하는 시스템**이다.

| 부문 | 점검 항목 | 자동화 |
|---|---|---|
| 1. 데이터 무결성 | 30일 곡선 결측(공백<48h), 핵심 파일 누적, epoch/이벤트 기록 | ✅ monthly_check |
| 2. 체결 품질 | 포지션 정합, 잔고 0/recon block 반복 없음, maker 비중·슬리피지 | ✅ (maker/슬립은 체결 누적 후) |
| 3. 성과 상태 | PSR>0.5, 드리프트 분위≥25%, 30일 내 CRITICAL 없음 (drift_history.csv 근거), 북별 편중 | ✅ |
| 4. 포트폴리오 구조 | 라이브 MDD>-40%, 북 간 상관 급등 없음, BB 역할 확인 | ✅ (상관은 분기 분석) |
| 5. 자가학습 판단 | 90일 미만 금지 / plateau·마진 / 쿨다운 / CRITICAL 예외 | ✅ _policy_block |

**실행 순서**: A. drift_status→equity→paper_state(epoch/halted/recon)→revalidation 확인
→ B. 정상=유지 / WARNING=후보 검토 / CRITICAL=예외 교체 / 운영 이상=학습 보류·복구 우선
→ C. 변경 없어도 self_learn_history.csv에 기록, 교체 시 다음 달까지 쿨다운.

판정 규칙: **부문 1·2(무결성·체결)에서 하나라도 실패 = "운영 이상 — 복구 우선"** (성과보다 생존).
